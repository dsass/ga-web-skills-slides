HTML
HyperText Markup Language
  • +HTML defines the structure and presentation of our web page
  • +We write HTML using blocks of elements, consisting of tags < >
  • +Some elements are used for describing and structuring text (e.g, <p>, <span>)
  • +While others are used to contain content (e.g, <div>)
HTML
anatomy of an HTML page 
            
              <!DOCTYPE html>
              <html>
                <head>
                  <title>A basic web page</title>
                </head>
                <body>
                  <div>
                    <span>Here is some text</span>
                  </div>
                </body>
              </html>

CSS
Cascading Style Sheets
  • +CSS is markup language that describes the style of an HTML document
  • +CSS is written to apply style to elements at different levels of specificity 
  •                   
                    /* Elements by tag */
                    p {
                      color: #000000;
                    }
                    
                    /* Elements by tag and class name (using the . identifier) */
                    div.content-container {
                      background: #121212;
                    }
                    
                    /* Elements by tag and id (using the # identifier) */
                    span#my-custom-span {
                      font-family: "Apercu-Pro", sans-serif;
                      font-size: 12px;
                    }
                    
                    /* Child elements of a specific element */
                    ul#my-custom-list > li {
                      list-style-type: none;
                    }
CSS
External & linked CSS
            
              <!-- External & Linked CSS-->
              <!DOCTYPE html>
              <html>
                <head>
                  <title>Welcome to My Website</title>
                  <link rel="stylesheet" href="css/style.css">
                </head>
                <!-- body, etc ... -->
              </html>
            
          
            
              /* css/style.css */
              .my-custom-div {
                background-color: #000000; 
                padding: 20px;
              }
              
              .my-custom-paragraph {
                color: #FFFFFF;

Git
What is Git?
  • +Git is a Version Control System (VCS) that tracks file changes in a directory
  • +You can think of git as a set of commands that help you track changes to your code
  • +The way we use these commands is via a GUI (graphical user interface), like GitHub Desktop
  • +Or with command line, using the CLI (command line interface)
GitHub
What is GitHub?
  • +GitHub is a hosting service for your code repositories
  • +GitHub comes with some features, like GitHub pages, which we'll use in an upcoming exercise
  • +There are other hosting services out there (Bitbucket, GitLab)
Git Basics I: Cloning your first repo
Basic terminology & commands
  • +We will start using Git by 'cloning' an existing repo
  • +A remote repo refers to a repository that is hosted somewhere (like GitHub)
  • +Cloning a repo refers to making a copy of it locally (downloaded to your machine)

Running a local server
Why do we need to do this?
  • +You should get in the habit of using a local server, even when working with static files
  • +You won't be able to access your file system (e.g, if you need to load in an image)
  • +You won't be able to make http requests, like when working with an API
Running a local server
2 ways
  • +Atom Live Server package, for Atom users
  • +Running in the terminal 
                      
                    # Check what version of Python is installed
                    python -V
                    
                    # above 3.X
                    python -m http.server 3000
                    
                    # above 2.X
                    python -m SimpleHTTPServer 3000
